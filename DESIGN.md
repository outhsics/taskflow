## Problem Statement

I want to build a personal task management app that I'll actually use daily. Existing apps fall into two categories: too simple (missing power features I need) or too complex (bloated with features I don't use, requires expensive subscriptions, or locks me into their cloud).

I need:
- **Power features**: Recurring tasks, categories/projects, priority levels, smart reminders
- **Full ownership**: Self-hosted, local-first data, no cloud dependency, easy export
- **Cross-platform access**: At minimum, web access from any device
- **AI-powered suggestions**: The 10x feature — learns my patterns and suggests tasks proactively

The research shows users abandon task apps when they become too complicated to maintain. The opportunity is **power through simplicity** — an app that FEELS simple but has intelligent AI features that work behind the scenes.

## What Makes This Cool

The "whoa" factor is the AI that actually learns you. Instead of you constantly configuring rules and settings, the app observes your patterns and makes intelligent suggestions:

- "I notice you always complete 'Weekly Report' on Friday afternoons. Should I create a recurring task?"
- "You've been postponing 'Review documentation' for 3 weeks. Want to bump the priority or remove it?"
- "Based on your completion rate, you tend to be most productive on Tuesday mornings. Here's your tasks for today."

This is AI as a thoughtful assistant, not a generic ChatGPT wrapper.

## Constraints

1. **Self-hosted**: Data must be stored locally, no cloud dependency
2. **Bring your own AI key**: User provides their own OpenAI/Anthropic API key
3. **MVP scope**: Web app first (mobile via browser acceptable), not native mobile apps
4. **Simple deployment**: Should be deployable to static hosting (GitHub Pages, Netlify) or run locally
5. **Ships fast**: Goal is working app in days, not weeks

## Premises

1. **Local browser storage is sufficient for personal use** — agree/disagree?
   - IndexedDB provides 50MB+ storage, enough for thousands of tasks
   - No backend complexity or database maintenance
   - Easy export/import as JSON for backup and migration

2. **Bring-your-own AI key is the right model** — agree/disagree?
   - User controls AI costs (no surprise charges)
   - No middleman — direct API calls to user's chosen provider
   - Privacy: User knows exactly what data is being sent to AI

3. **Natural language task creation is a power feature** — agree/disagree?
   - "Call mom every Sunday at 6pm" should create a recurring task automatically
   - Requires parsing but significantly reduces friction for data entry
   - If parsing fails, gracefully fall back to manual form fields

4. **AI suggestions should be opt-in, never intrusive** — agree/disagree?
   - AI analyzes patterns in the background
   - Suggestions appear as dismissible prompts
   - User can disable AI suggestions entirely if they prefer

## Approaches Considered

### Approach A: Minimal Viable (CHOSEN)

**Summary:** Single-page web app with local browser storage, Bring Your Own AI Key, and core power features. Ships in days.

**Tech Stack:**
- Frontend: React + TypeScript + Vite (fast, modern, great DX)
- Storage: IndexedDB via Dexie.js (reliable, 50MB+ capacity)
- AI: OpenAI API (user-provided key, stored locally)
- Natural Language: Custom parser + GPT-4 fallback
- Deployment: Static hosting (GitHub Pages, Netlify) or local `npm run dev`

**Effort:** S (small) — ~2-4 days with CC+gstack
**Risk:** Low — simple architecture, easy to iterate

**Pros:**
- No backend to maintain — runs entirely in the browser
- Data stored locally — fully self-hosted
- Bring your own API key — you control AI costs
- Can be deployed to GitHub Pages or any static hosting
- Easy to export/import data as JSON
- Fast iteration loop

**Cons:**
- No sync between devices (manual export/import required)
- IndexedDB has limits (~50-100MB depending on browser)
- AI features require internet and user's API key
- No native mobile experience (mobile browser only)

**Reuses:** Standard web stack, proven patterns, no custom infrastructure

### Approach C: Git-Based Tasks

**Summary:** Tasks stored as markdown files in a git repo, with CLI + web UI.

**Tech Stack:**
- Storage: Markdown files in git repository
- CLI: Node.js CLI tool
- Web UI: Simple browser interface
- Sync: Git (GitHub/GitLab or self-hosted)

**Effort:** M (medium) — ~1 week with CC+gstack
**Risk:** Medium — unconventional approach, but highly flexible

**Pros:**
- Data as plain text markdown — max portability
- Git version control for task history
- CLI for power users, web UI for casual use
- Can sync across devices via git

**Cons:**
- Git operations add friction for non-technical users
- Merge conflicts possible (though rare for single-user)

- Different mental model than traditional task apps

**Why not chosen:** Cool idea, but adds unnecessary complexity for a personal task manager. The goal is productivity, not infrastructure.

## Recommended Approach: Minimal Viable (Approach A)

**Core Features (MVP):**

1. **Task Management**
   - Create, edit, delete tasks
   - Mark tasks complete/incomplete
   - Task title, description, due date/time
   - Priority levels: Normal, Important, Urgent (color-coded)
   - Categories/Projects with color coding

2. **Recurring Tasks**
   - Daily, weekly, monthly patterns
   - Custom intervals (e.g., "every 2 weeks")
   - Next occurrence auto-creates on completion
   - Visual indicator for recurring tasks

3. **Smart Reminders**
   - Browser notifications (with permission)
   - Due date/time alerts
   - Snooze functionality
   - Quiet hours configuration

4. **Views & Filtering**
   - All Tasks view grouped by category
   - Today view (tasks due today)
   - This Week view (next 7 days)
   - Recurring view (all recurring tasks)
   - Completed view (history)
   - Filter by category, priority, completion status
   - Sort by due date, priority, creation date

5. **Natural Language Task Creation**
   - Simple parser for common patterns:
     - "Call mom every Sunday at 6pm" → recurring task
     - "Submit report by Friday 5pm" → due date
     - "URGENT: fix production bug" → priority
   - Fallback to manual form if parsing fails
   - Shows parsed interpretation for confirmation

6. **AI-Powered Suggestions**
   - Pattern recognition:
     - Tasks completed at same time/day
     - Tasks postponed multiple times
     - Productivity patterns by day/time
   - Proactive suggestions (dismissible):
     - "Should I make 'Weekly Report' a recurring task?"
     - "You've postponed 'Review docs' 3 times. Bump priority?"
   - Smart task suggestions based on context
   - User can disable AI entirely

7. **Data Ownership**
   - All data stored in IndexedDB (local browser storage)
   - Export to JSON (full backup)
   - Import from JSON (restore or migrate)
   - API key stored locally (never sent to any server except AI provider)
   - No analytics, no tracking, no cloud dependency

8. **Settings**
   - Configure OpenAI API key (or Anthropic)
   - Enable/disable AI suggestions
   - Set quiet hours for reminders
   - Choose theme (light/dark)
   - Clear all data (with confirmation)

**Implementation Order:**

1. **Day 1**: Core task CRUD + IndexedDB storage + basic UI
2. **Day 2**: Categories, priorities, recurring tasks, views
3. **Day 3**: Natural language parsing, reminders
4. **Day 4**: AI integration (pattern analysis + suggestions)

**Data Model (Simplified):**

```typescript
interface Task {
  id: string;              // UUID
  title: string;
  description?: string;
  completed: boolean;
  priority: 'normal' | 'important' | 'urgent';
  categoryId: string;
  dueDate?: Date;
  recurring?: {            // null if one-time task
    frequency: 'daily' | 'weekly' | 'monthly' | 'custom';
    interval?: number;     // for custom: every N days/weeks
    daysOfWeek?: number[]; // for weekly: [0,6] = Sun, Sat
    dayOfMonth?: number;   // for monthly: 1-31
    endDate?: Date;        // optional: stop recurring after this date
    occurrenceCount?: number; // optional: stop after N completions
    forever?: boolean;     // true = never stop (default)
    // Precedence: forever=true → infinite. If forever=false/unset, occurrenceCount takes priority over endDate.
    // If both endDate and occurrenceCount are set, whichever occurs first stops the recurrence.
  };
  createdAt: Date;
  completedAt?: Date;
  nextOccurrence?: Date;   // for recurring tasks
}

interface Category {
  id: string;              // UUID
  name: string;
  color: string;           // hex color
  icon?: string;           // emoji
}

interface AppSettings {
  aiProvider: 'openai' | 'anthropic';
  apiKey: string;
  aiEnabled: boolean;
  quietHoursStart?: string; // HH:MM format
  quietHoursEnd?: string;
  theme: 'light' | 'dark';
}
```

**AI Integration Design:**

1. **Pattern Analysis (runs daily or on demand)**
   - Fetch all completed tasks from IndexedDB
   - Send anonymized data to OpenAI API:
     ```json
     {
       "tasks": [
         {"title": "Submit weekly report", "completedAt": "2025-03-15T17:00:00Z"},
         {"title": "Submit weekly report", "completedAt": "2025-03-22T17:00:00Z"}
       ]
     }
     ```
   - Prompt: "Analyze these completed tasks. Identify patterns in timing, frequency, and relationships. Suggest recurring tasks or optimizations."

2. **Smart Suggestions (context-aware)**
   - Triggers:
     - On first app open after 4+ hours of inactivity (debounced, not on tab switching)
     - Manual "Get suggestions" button click (always available)
     - Not triggered on rapid reopens within 10 minutes
   - Analysis scope:
     - Tasks completed in past 30 days (pattern detection)
     - Tasks due today/tomorrow (urgency)
     - Overdue tasks (attention needed)
     - Tasks postponed 3+ times (consider removal or priority change)
   - Suggestion types:
     - "Should 'X' be a recurring task?" (based on completion pattern)
     - "Consider increasing priority of 'X'" (postponed multiple times)
     - "You're most productive on Tuesdays" (based on completion history)
     - "Tasks overdue: [list]" (simple reminder)
   - User actions: Accept, Dismiss, Disable AI suggestions

3. **Natural Language Parsing**
   - First pass: Regex-based parser for common patterns
     - Recurring: `/(every|daily|weekly|monthly)/i`
     - Time: `/\b(at|by)\s+(\d{1,2}(?::\d{2})?\s*(?:am|pm)?)\b/i`
     - Priority: `/(urgent|important|asap)/i`
   - Fallback: Send to GPT-4 for complex parsing
   - Return structured data: `{ title, priority, dueDate, recurring }`

**Privacy & Security:**

- API key stored in localStorage, never sent except to AI provider
- Task data only sent to AI provider when user interacts with AI features
- User can clear all data instantly (Settings → Clear All Data)
- No analytics, no telemetry, no third-party trackers

**Data Migration & Error Handling:**

1. **Schema versioning**
   - Store schema version in IndexedDB metadata
   - On app load, check version; if outdated, run migration
   - Migrations are incremental: v1 → v2 → v3 (not v1 → v3 directly)
   - Each migration includes: version number, description, upgrade function, rollback function
   - On migration failure: automatically rollback to previous version, alert user with error details
   - Store last successful migration timestamp for recovery

3. **Error handling**
   - IndexedDB quota exceeded: Show error with "Export your data to free up space" CTA
   - Corrupted data: Attempt recovery from backup, show error with "Last good backup: [date]"
   - API key invalid: Show "Invalid API key" error in settings, clear key
   - AI rate limit: Show "Too many AI requests. Try again in [time]" message

4. **Backup strategy**
   - MVP: Manual export only (Settings → Export)
   - Automatic backups: Consider for v2 (requires Service Worker or File System Access API)
   - Before any schema migration, auto-export current data to downloads folder

**Deployment Options:**

1. **Local development**: `npm run dev` — runs on localhost
2. **Static hosting**: Build with `npm run build`, deploy to:
   - GitHub Pages (free, private repos)
   - Netlify (free tier)
   - Vercel (free tier)
   - Any web server
3. **Self-hosted**: Run build artifacts on any web server

## Open Questions

1. **Natural language parsing complexity**: How sophisticated should the parser be?
   - Simple: regex for common patterns, fallback to manual form
   - Complex: GPT-4 for all parsing, slower but more accurate
   - **Decision**: Start with regex + GPT fallback, measure accuracy

2. **AI frequency**: How often should AI analyze patterns?
   - Every time app opens (might be slow/expensive)
   - Daily background job (requires service worker)
   - Manual "Analyze my patterns" button
   - **Decision**: Manual button first, add periodic analysis later if useful

3. **Reminder persistence**: Browser notifications don't persist across sessions
   - Show reminders only when app is open
   - Use Service Worker for background notifications (complex)
   - **Decision**: App-open reminders first, evaluate if background needed
